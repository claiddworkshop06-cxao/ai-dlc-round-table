# INT-000 User Stories

## 備品マスタ管理

### US-001: 備品の新規登録
管理者として、備品名・説明・デフォルト返却期限日数を入力して備品を登録したい。
なぜなら、貸出管理の対象となる備品を一元的に管理する必要があるから。

受入基準:
- 備品名（必須）、説明（任意）、デフォルト返却期限日数（任意）を入力して登録できる
- 登録後、備品一覧に反映される
- 備品名が未入力の場合はバリデーションエラーを表示する

### US-002: 備品の編集・削除
管理者として、登録済みの備品情報を編集・削除したい。
なぜなら、備品情報の誤りを修正したり、廃棄した備品を管理対象から除外したいから。

受入基準:
- 備品一覧から任意の備品を選択して編集フォームを開ける
- 備品名・説明・デフォルト返却期限日数を変更して保存できる
- 貸出中でない備品を削除できる
- 削除後、備品一覧から該当備品が消える

### US-003: 備品一覧の確認
管理者として、登録されている備品の一覧と各備品の貸出状況を確認したい。
なぜなら、どの備品が現在貸出中か、どの備品が利用可能かを把握したいから。

受入基準:
- 備品一覧ページで全備品が表示される
- 各備品の貸出状況（利用可能/貸出中）がわかる
- 返却期限超過・期限間近の備品がハイライト表示される

## QRコード管理

### US-004: QRコードの発行・表示
管理者として、備品ごとのQRコードを生成・表示したい。
なぜなら、備品にQRコードシールを貼り付けて利用者がスキャンできるようにしたいから。

受入基準:
- 備品詳細ページにQRコードが表示される
- QRコードをスキャンすると該当備品の詳細ページURLに遷移する
- QRコードを画面上に大きく表示して印刷・撮影できる

## 貸出・返却操作

### US-005: QRコードスキャンによる貸出
利用者として、備品のQRコードをスキャンして貸出申請をしたい。
なぜなら、手書き台帳や担当者への声がけなしに、自分で貸出操作を完結させたいから。

受入基準:
- スマートフォンのカメラでQRコードをスキャンできる
- スキャン後に備品詳細ページが表示される
- 備品が「利用可能」の場合、借り手名と返却予定日を入力して「借りる」ボタンで貸出できる
- 貸出後、備品の状態が「貸出中」に更新される

### US-006: QRコードスキャンによる返却
利用者として、備品のQRコードをスキャンして返却操作をしたい。
なぜなら、担当者不在でも返却を自己申告で完結させたいから。

受入基準:
- 備品が「貸出中」の場合、「返却する」ボタンが表示される
- ボタンタップで返却記録が作成され、備品の状態が「利用可能」に更新される

## 状況確認・履歴

### US-007: 貸出状況の一覧確認
管理者・利用者として、現在の全備品の貸出状況を一覧で確認したい。
なぜなら、特定の備品が使えるかどうかを事前に確認したいから。

受入基準:
- 貸出状況一覧ページで全備品のステータスが確認できる
- 貸出中の備品について借り手名・返却予定日が表示される
- 期限超過の備品は赤、期限間近（1日以内）は黄色でハイライト表示される

### US-008: 貸出履歴の確認
管理者として、備品ごとの貸出・返却履歴を確認したい。
なぜなら、過去にどの備品が誰に貸し出されたかを追跡したいから。

受入基準:
- 備品詳細ページに貸出履歴（借り手名・貸出日・返却日）が表示される
- 全備品の履歴を日付降順で一覧表示できるページがある
